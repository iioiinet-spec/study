인접 행렬 DFS/BFS 대화형 학습

인접 행렬이란?

정점 간 연결 정보를 2차원 배열에 저장하는 방식.
graph[a][b] = 1이면 a와 b가 연결됨.

무방향 그래프

a↔b이면 graph[a][b] = 1과 graph[b][a] = 1을 동시에 저장해야 함.

DFS (깊이 우선)

한 방향으로 끝까지 파고들었다가 돌아오는 방식.
재귀 함수로 구현.

BFS (너비 우선)

가까운 정점부터 넓게 탐색하는 방식.
큐(queue)로 구현.

예제 그래프 (정점 5개, 간선 5개)

1-2, 1-3, 2-4, 2-5, 3-5 연결

1-21-32-42-53-5

visited[ ] 배열이 필요한 이유

그래프는 순환이 가능해서, 방문 기록 없이 탐색하면 1→2→1→2→... 무한 반복이 됩니다. visited[i] = 1로 체크해 방지합니다.

아래 인접 행렬에서 파란색 셀(1)은 두 정점이 연결되어 있음을 뜻합니다. 무방향이므로 대각선 기준으로 대칭입니다.

graph[a][b] — 행: 출발, 열: 도착
입력 코드
for (int i = 0; i < m; i++) { int a, b; scanf("%d %d", &a, &b); graph[a][b] = 1; graph[b][a] = 1; // 무방향! }

이웃 정점 찾기

현재 정점 now에서 연결된 정점을 찾으려면 1번부터 n번까지 전부 확인합니다:

for (int next = 1; next <= n; next++) { if (graph[now][next] == 1) { // now와 next가 연결됨 } }

DFS — 깊이 우선

  • 재귀 함수 사용
  • 한 방향으로 끝까지 파고듦
  • 방문 즉시 출력, 즉시 재귀
  • 콜 스택이 사용됨
  • 미로 찾기, 경로 탐색에 유용

BFS — 너비 우선

  • 큐(배열) 사용
  • 가까운 것부터 넓게 탐색
  • 큐에 넣을 때 방문 처리
  • 큐 배열이 사용됨
  • 최단 경로 탐색에 유용

자주 하는 실수

무방향인데 한쪽만 저장

graph[a][b] = 1만 넣고 graph[b][a] = 1을 빠뜨리는 실수.

DFS 후 visited 초기화 안 함

BFS 전에 visited[i] = 0으로 모두 초기화해야 합니다.

BFS에서 큐 초기화 안 함

front = rear = 0으로 다시 설정해야 합니다.

BFS 방문 처리를 늦게 함

큐에 넣을 때 바로 visited = 1 해야 합니다. 꺼낼 때 하면 중복 삽입 발생.
