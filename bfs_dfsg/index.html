격자 BFS/DFS 대화형 학습

격자 = 그래프

각 칸을 정점으로, 상하좌우 이동 가능하면 간선이 있다고 생각합니다. 2차원 배열 위에서 BFS/DFS를 수행합니다.

대표 문제 유형

  • 미로 길 찾기
  • 섬/영역 개수 세기
  • 최단거리 구하기
  • 도달 가능한 칸 찾기

방향 배열 — 상하좌우 이동의 핵심

현재 위치 (x, y)에서 네 방향을 이렇게 계산합니다: nx = x + dx[i], ny = y + dy[i]

위
(x-1,y)
좌
(x,y-1)
현재
(x,y)
우
(x,y+1)
아래
(x+1,y)
dx[4] = {-1, 1, 0, 0}; dy[4] = {0, 0, -1, 1};

이동 전 반드시 확인할 3가지

if (nx < 0 || ny < 0 || nx >= n || ny >= m) continue; // ① 범위 초과 if (map[nx][ny] == '0') continue; // ② 벽 if (visited[nx][ny]) continue; // ③ 이미 방문

문자열형 (붙어서 입력)

예: 11010

char map[MAX][MAX]로 저장.
비교할 때 반드시 '1', '0' (문자) 사용.

숫자형 (띄어쓰기로 입력)

예: 1 1 0 1 0

int map[MAX][MAX]로 저장.
비교할 때 숫자 1, 0 사용.

// 문자열형 격자 입력 char map[MAX][MAX]; int visited[MAX][MAX]; int n, m; scanf("%d %d", &n, &m); for (int i = 0; i < n; i++) { scanf("%s", map[i]); // 한 줄 통째로 }

DFS — 깊이 우선

  • 재귀 함수 사용
  • 한 방향으로 끝까지 파고듦
  • 방문 즉시 재귀 호출
  • 영역 개수 세기에 자주 사용
  • 스택(콜 스택) 사용

BFS — 너비 우선

  • 큐(배열) 사용
  • 가까운 칸부터 넓게 탐색
  • 큐 삽입 시 방문 처리
  • 최단거리 구할 때 사용
  • dist[][]와 조합 가능
자주 하는 실수

범위 검사 누락

nx, ny가 배열 범위를 벗어나면 런타임 에러 발생. 반드시 nx<0||ny<0||nx>=n||ny>=m 체크.

문자 격자에서 숫자로 비교

map[x][y]==1(틀림) → map[x][y]=='1'(맞음). char 배열은 문자로 비교.

BFS에서 큐 삽입 시 방문 처리 안 함

꺼낼 때 처리하면 같은 칸이 큐에 여러 번 삽입되어 중복 방문 발생.

visited[][] 초기화 누락

전역 배열은 자동으로 0이지만, 여러 번 실행하는 경우 반드시 초기화 확인.
