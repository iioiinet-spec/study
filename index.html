BFS — 너비 우선 탐색

중등정보올림피아드 핵심 탐색 알고리즘 완전 정복

BFS란?
Breadth-First Search — 너비 우선 탐색. 가까운 곳부터 넓게 퍼지며 탐색하는 방법입니다.

돌을 물에 던지면 파장이 가까운 곳부터 점점 멀리 퍼집니다. BFS도 똑같습니다 — 시작점에서 거리 1, 2, 3 순서로 탐색합니다.

1
시작점을 큐에 넣고 방문 처리한다
2
큐에서 하나 꺼낸다
3
연결된 아직 방문하지 않은 정점을 모두 큐에 넣는다
4
큐가 빌 때까지 2~3을 반복한다

왜 큐를 쓰는가?
큐는 먼저 들어간 것이 먼저 나오는 FIFO 구조입니다. 먼저 발견한 정점을 먼저 탐색해야 거리 순서가 유지됩니다.
push — 뒤에 넣기 pop — 앞에서 꺼내기 front — 맨 앞 확인 empty — 비었는지 확인
BFS가 잘 쓰이는 문제 유형
최단거리·최소이동 미로 탈출 방문 순서 바이러스·불 전파 연결 요소 개수 섬 개수

문제에서 "최소 이동 횟수", "몇 단계 만에 도달", "동시에 퍼짐"이 보이면 BFS 신호!

BFS 의사코드
BFS(시작점): 시작점을 큐에 넣는다 visited[시작점] = 1 while 큐가 비어 있지 않다면: now = 큐에서 꺼낸다 now와 연결된 모든 next에 대해: if visited[next] == 0: visited[next] = 1 큐에 next 넣는다

방문 처리(visited = 1)는 큐에 넣을 때 합니다. 꺼낼 때 하면 같은 정점이 여러 번 삽입될 수 있습니다!

왜 BFS가 최단거리를 보장할까?
BFS는 거리 1 → 거리 2 → 거리 3 순서로 탐색합니다. 어떤 정점을 처음 방문했다는 것은 그 정점에 도달하는 가장 짧은 경로로 도착했다는 뜻입니다.

단, 모든 간선의 비용이 같을 때(한 번 이동 = 비용 1) 성립합니다. 중등올림피아드에서는 대부분 이 경우입니다.

// 최단거리 BFS 핵심 추가 부분 int dist[MAX] = {0}; dist[start] = 0; if (graph[now][next] == 1 && visited[next] == 0) { visited[next] = 1; dist[next] = dist[now] + 1; // 핵심! queue[rear++] = next; }

기본 BFS C코드
#include <stdio.h> #define MAX 101 int graph[MAX][MAX]; int visited[MAX]; int queue[MAX]; int front = 0, rear = 0; int n, m; void bfs(int start) { queue[rear++] = start; visited[start] = 1; while (front < rear) { int now = queue[front++]; printf("%d ", now); for (int next = 1; next <= n; next++) { if (graph[now][next] == 1 && visited[next] == 0) { visited[next] = 1; queue[rear++] = next; } } } } int main() { scanf("%d %d", &n, &m); for (int i = 0; i < m; i++) { int a, b; scanf("%d %d", &a, &b); graph[a][b] = graph[b][a] = 1; } bfs(1); return 0; }
최단거리 BFS C코드
int dist[MAX]; void bfs(int start) { queue[rear++] = start; visited[start] = 1; dist[start] = 0; // 시작점 거리 = 0 while (front < rear) { int now = queue[front++]; for (int next = 1; next <= n; next++) { if (graph[now][next] == 1 && visited[next] == 0) { visited[next] = 1; dist[next] = dist[now] + 1; // 핵심! queue[rear++] = next; } } } }
코드 핵심 요소 해설
1
graph[a][b]=1 — a와 b가 연결됨
2
queue[rear++]=start — 큐 뒤에 삽입
3
while(front<rear) — 큐가 비어 있지 않은 동안
4
queue[front++] — 큐 앞에서 꺼내기
5
visited[next]=1 — 큐에 넣을 때 즉시 방문 처리
6
dist[next]=dist[now]+1 — 최단거리 갱신
BFS에서 가장 많이 하는 실수 6가지
실수 1
방문 처리를 꺼낼 때 하는 실수
큐에 넣을 때 바로 visited = 1 해야 중복 삽입을 막습니다.
실수 2
인덱스 범위 체크 누락
미로 문제에서 nx, ny가 배열 범위를 벗어나는지 먼저 확인해야 합니다.
실수 3
양방향인지 단방향인지 혼동
무방향 그래프면 graph[a][b]와 graph[b][a] 둘 다 처리해야 합니다.
실수 4
방문 배열 초기화 누락
여러 테스트 케이스가 있을 때 특히 조심해야 합니다.
실수 5
최단거리 배열 초기화 실수
시작점 dist = 0, 나머지는 미방문 상태로 구분해야 합니다.
실수 6
front와 rear 혼동
front++로 꺼내고, rear++로 넣습니다. front < rear이면 큐에 원소가 있습니다.
