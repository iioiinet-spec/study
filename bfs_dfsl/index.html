인접 리스트 DFS/BFS 대화형 학습

인접 행렬의 한계

정점이 100개일 때, 정점 1이 2개와만 연결되어 있어도 for(next=1; next<=100)을 전부 돌아야 합니다. 비어 있는 칸도 확인해야 해서 비효율적입니다.

인접 리스트의 장점

실제로 연결된 정점만 저장합니다. 정점 1에 2개가 연결되어 있으면 2번만 확인하면 됩니다. 크고 희소한 그래프에서 효율적입니다.

예제 그래프 — 간선: 1-2, 1-3, 2-4, 2-5, 3-5

인접 리스트로 표현하면:
1: [2, 3] 2: [1, 4, 5] 3: [1, 5] 4: [2] 5: [2, 3]

DFS/BFS에서 바뀌는 것은 딱 하나

인접 행렬 — 전부 확인
for (next=1; next<=n; next++) { if (graph[now][next])... }
인접 리스트 — 연결된 것만
for (i=0; i<cnt[now]; i++) { next = adj[now][i]; }

배열형 인접 리스트 선언

int adj[MAX][MAX]; // adj[x][i] = x번 정점의 i번째 연결 정점 int cnt[MAX]; // cnt[x] = x번 정점의 연결 개수 int visited[MAX]; // 방문 여부

간선 삽입 코드

adj[a][cnt[a]++] = b; // a의 목록 맨 뒤에 b 삽입 adj[b][cnt[b]++] = a; // b의 목록 맨 뒤에 a 삽입 (무방향)

인접 행렬

  • graph[a][b] 사용
  • 이웃 찾을 때 1~n 전부 확인
  • 구조가 직관적, 이해 쉬움
  • 메모리: O(n²)
  • 이웃 탐색: O(n)
  • 작은 문제 설명용에 좋음

인접 리스트

  • adj[a][i], cnt[a] 사용
  • 이웃 찾을 때 연결된 것만 확인
  • 코드가 조금 더 복잡
  • 메모리: O(n+m)
  • 이웃 탐색: O(degree)
  • 큰 문제, 실전 코딩에 유리

자주 하는 실수

무방향인데 한쪽만 삽입

adj[a][cnt[a]++]=b 와 adj[b][cnt[b]++]=a 를 반드시 둘 다 해야 합니다.

cnt 배열 관리 실수

각 정점에 몇 개가 들어갔는지 cnt[x]로 정확히 추적해야 합니다.

정렬 안 해서 방문 순서 오류

작은 번호부터 방문 조건이 있으면 입력 후 각 adj[i]를 오름차순 정렬해야 합니다.

DFS 후 visited/큐 초기화 안 함

BFS 전에 visited[i]=0, front=rear=0 으로 반드시 초기화해야 합니다.
