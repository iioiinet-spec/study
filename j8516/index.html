정올 #8516 점프점프

오른쪽 건물로만 이동하고, 높이 차이가 K 이하인 건물로 점프할 수 있을 때 각 건물에 도달 가능한지 Fenwick Tree와 좌표 압축으로 빠르게 판단합니다.

문제 핵심 단순 풀이 핵심 아이디어 좌표 압축 Fenwick Tree 예제 설명 정답 코드 시뮬레이터 퀴즈

1. 문제 핵심

정올이는 1번 건물에서 시작합니다. 오른쪽 건물로만 이동할 수 있고, 두 건물의 높이 차이가 K 이하여야 합니다.
1. 현재 건물의 높이: h[i]
2. 이동하려는 건물의 높이: h[j]
3. 이동 방향 조건: j > i
4. 높이 조건: |h[i] - h[j]| ≤ K

j > i 그리고 |h[i] - h[j]| ≤ K

입력 범위는 N ≤ 200000, K ≤ 10⁹ 이므로 빠른 알고리즘이 필요합니다.

2. 단순 풀이가 안 되는 이유

2.1 모든 건물 쌍 검사

for (int i = 1; i <= N; i++) {
    for (int j = i + 1; j <= N; j++) {
        ...
    }
}

이 방식은 모든 i, j 쌍을 확인하므로 O(N²)입니다.

2.2 N = 200000이면?

O(N²)
약 40,000,000,000번 수준의 비교가 될 수 있습니다.
필요한 방향
각 건물마다 빠르게 “올 수 있는 이전 건물”이 있는지만 확인해야 합니다.

3. 핵심 아이디어

i번 건물에 도달 가능한지 알고 싶다면, 왼쪽에 있는 도달 가능한 건물 중 높이가 h[i]-K 이상 h[i]+K 이하인 건물이 하나라도 있으면 됩니다.

h[i] - K ≤ 이전 건물 높이 ≤ h[i] + K

왼쪽 건물만 본다
→
도달 가능한 건물만 본다
→
높이 범위 안에 있는지 확인
→
하나라도 있으면 가능

3.1 판단 문장

현재 건물에 올 수 있는 이전 건물이 존재하는가?

4. 왜 좌표 압축을 쓰는가?

높이는 최대 10⁹입니다. 높이를 그대로 배열 인덱스로 쓰면 배열이 너무 커집니다.

실제로 등장한 높이만 정렬해서 번호를 붙이면 작은 인덱스로 관리할 수 있습니다.

5. Fenwick Tree에 저장하는 것

Fenwick Tree에는 “도달 가능한 건물의 높이”를 저장합니다.
처음
1번 건물은 항상 도달 가능입니다.
possible[1] = 1;
update(pos, 1);
확인
현재 높이 범위 안에 도달 가능한 건물이 있는지 봅니다.
if (range_sum(left, right) > 0)
가능하면 추가
현재 건물도 도달 가능 높이로 Fenwick Tree에 넣습니다.
update(idx, 1);

5.1 Fenwick Tree 기본 함수

update, sum, range_sum 코드 보기 
void update(int idx, int value) {
    while (idx <= N) {
        bit[idx] += value;
        idx += idx & -idx;
    }
}

int sum(int idx) {
    int result = 0;

    while (idx > 0) {
        result += bit[idx];
        idx -= idx & -idx;
    }

    return result;
}

int range_sum(int left, int right) {
    if (left > right) return 0;
    return sum(right) - sum(left - 1);
}

5.2 lower_bound와 upper_bound

높이 범위 [h[i]-K, h[i]+K]를 압축 번호 범위로 바꾸기 위해 이분 탐색을 사용합니다.

left = lower_bound(sorted, low)

right = upper_bound(sorted, high) - 1

6. 예제 설명

6.1 입력

5 4 8 7 2

6.2 단계별 판단

자세한 풀이 보기

1번 건물 높이 5는 시작점이므로 가능합니다.

2번 건물 높이 4: |5 - 4| = 1이므로 가능합니다.

3번 건물 높이 8: 5와 차이 3, 4와 차이 4이므로 불가능합니다.

4번 건물 높이 7: |5 - 7| = 2이므로 가능합니다.

5번 건물 높이 2: |4 - 2| = 2이므로 가능합니다.

출력: 1 1 0 1 1

7. 정답 C 코드

#include <stdio.h>
#include <stdlib.h>

#define MAXN 200005

int N;
long long K;
long long h[MAXN];
long long sorted[MAXN];

int bit[MAXN];

int compare_ll(const void *a, const void *b) {
    long long x = *(long long *)a;
    long long y = *(long long *)b;

    if (x < y) return -1;
    if (x > y) return 1;
    return 0;
}

void update(int idx, int value) {
    while (idx <= N) {
        bit[idx] += value;
        idx += idx & -idx;
    }
}

int sum(int idx) {
    int result = 0;

    while (idx > 0) {
        result += bit[idx];
        idx -= idx & -idx;
    }

    return result;
}

int range_sum(int left, int right) {
    if (left > right) return 0;
    return sum(right) - sum(left - 1);
}

int lower_bound_idx(long long arr[], int size, long long target) {
    int left = 1;
    int right = size + 1;

    while (left < right) {
        int mid = (left + right) / 2;

        if (arr[mid] >= target) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }

    return left;
}

int upper_bound_idx(long long arr[], int size, long long target) {
    int left = 1;
    int right = size + 1;

    while (left < right) {
        int mid = (left + right) / 2;

        if (arr[mid] > target) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }

    return left;
}

int main() {
    scanf("%d %lld", &N, &K);

    for (int i = 1; i <= N; i++) {
        scanf("%lld", &h[i]);
        sorted[i] = h[i];
    }

    qsort(sorted + 1, N, sizeof(long long), compare_ll);

    int possible[MAXN] = {0};

    possible[1] = 1;

    int pos = lower_bound_idx(sorted, N, h[1]);
    update(pos, 1);

    for (int i = 2; i <= N; i++) {
        long long low = h[i] - K;
        long long high = h[i] + K;

        int left = lower_bound_idx(sorted, N, low);
        int right = upper_bound_idx(sorted, N, high) - 1;

        if (range_sum(left, right) > 0) {
            possible[i] = 1;

            int idx = lower_bound_idx(sorted, N, h[i]);
            update(idx, 1);
        }
    }

    for (int i = 1; i <= N; i++) {
        printf("%d ", possible[i]);
    }

    return 0;
}

11. 최종 정리

느린 생각
모든 이전 건물과 현재 건물을 비교한다.
빠른 생각
도달 가능한 이전 높이가 현재 높이 ± K 범위에 있는지만 확인한다.
좌표 압축
큰 높이를 작은 번호로 바꾼다.
Fenwick Tree
범위 안에 도달 가능한 높이가 있는지 빠르게 구한다.
한 줄 핵심: 현재 건물에 올 수 있는 이전 건물이 존재하는지만 Fenwick Tree로 빠르게 확인한다.
