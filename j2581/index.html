정올 #2581 예산

총예산을 넘지 않으면서 가능한 가장 큰 상한액을 찾는 이분 탐색 문제를 직접 움직이며 이해해 봅시다.

1. 문제를 쉽게 이해하기

여러 지방이 예산을 신청했습니다. 하지만 전체 예산은 정해져 있습니다. 모두 원하는 만큼 줄 수 없으면 상한액을 정합니다.

신청 예산이 상한액보다 크면 → 상한액만 준다.
신청 예산이 상한액보다 작거나 같으면 → 신청한 만큼 준다.

초등학생 버전으로 말하면, 아이들이 사탕을 많이 달라고 했지만 선생님 사탕이 부족해서 “한 사람당 최대 몇 개까지만 줄게!”라고 규칙을 정하는 것입니다.

2. 직접 입력해서 이분 탐색 보기

처음 작은값 left

가장 작게는 모두 0원씩 줄 수 있습니다.

0

처음 큰값 right

가장 큰 신청 예산보다 클 필요는 없습니다.

150

정답 상한액

가능한 가장 큰 상한액입니다.

127

3. 현재 단계 자세히 보기

분석하기를 누르면 이분 탐색 과정이 나옵니다.

L 0
M 75
R 150

현재 상한액으로 나누어 주면?

현재 mid

75

300

4. 이분 탐색 전체 과정 표

단계 left right mid 총합 결과 다음 행동

5. 왜 처음 작은값과 큰값을 이렇게 잡을까?

  1. left = 0입니다. 모두 0원씩 주는 것은 항상 가능하기 때문입니다.
  2. right = 가장 큰 신청 예산입니다. 아무도 그보다 많이 신청하지 않았으므로 상한액을 더 크게 할 필요가 없습니다.
  3. 예를 들어 신청 예산이 120, 110, 140, 150이면 가장 큰 값은 150입니다.
  4. 그래서 처음 탐색 범위는 0부터 150까지입니다.
  5. 그 가운데 숫자인 mid를 시험해 보고, 가능하면 더 크게, 불가능하면 더 작게 범위를 줄입니다.

6. 127이 나오는 과정

신청 예산이 120, 110, 140, 150이고 총예산이 485일 때를 봅시다.

  1. 모두 원하는 만큼 주면 120 + 110 + 140 + 150 = 520입니다. 485를 넘으므로 상한액이 필요합니다.
  2. 처음 범위는 0부터 150입니다. 0은 가장 작은 가능값, 150은 가장 큰 신청 예산입니다.
  3. 상한액 125를 시험하면 120 + 110 + 125 + 125 = 480입니다. 가능하므로 더 크게 해봅니다.
  4. 상한액 130을 시험하면 120 + 110 + 130 + 130 = 490입니다. 불가능하므로 줄여야 합니다.
  5. 상한액 127을 시험하면 120 + 110 + 127 + 127 = 484입니다. 가능합니다.
  6. 상한액 128을 시험하면 120 + 110 + 128 + 128 = 486입니다. 불가능합니다.
  7. 따라서 127은 가능하고 128은 불가능하므로 정답은 127입니다.

7. 그냥 총예산을 지방 수로 나누면 안 되는 이유

485를 4개 지방으로 나누면 약 121입니다.

485 ÷ 4 = 121.25

상한액을 121로 하면:

120 + 110 + 121 + 121 = 472

예산이 13이나 남습니다. 왜냐하면 120과 110을 신청한 지방은 121까지 받을 필요가 없기 때문입니다. 그래서 단순 나누기가 아니라, 가능한 상한액을 계속 시험해야 합니다.

8. 정답 C 코드

#include <stdio.h>

int main() {
    int N;
    int budget[10000];
    int M;
    int i;

    int max_budget = 0;
    int left, right, mid;
    int answer = 0;

    scanf("%d", &N);

    for (i = 0; i < N; i++) {
        scanf("%d", &budget[i]);

        if (budget[i] > max_budget) {
            max_budget = budget[i];
        }
    }

    scanf("%d", &M);

    left = 0;
    right = max_budget;

    while (left <= right) {
        long long sum = 0;

        mid = (left + right) / 2;

        for (i = 0; i < N; i++) {
            if (budget[i] > mid) {
                sum += mid;
            } else {
                sum += budget[i];
            }
        }

        if (sum <= M) {
            answer = mid;
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    printf("%d\n", answer);

    return 0;
}
