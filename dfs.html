DFS — 깊이 우선 탐색

중등정보올림피아드 핵심 탐색 알고리즘 — 한 방향으로 끝까지, 막히면 되돌아오기

DFS란?
Depth-First Search — 깊이 우선 탐색. 한 방향으로 가능한 만큼 끝까지 들어간 뒤, 더 이상 갈 수 없으면 되돌아와서 다른 길을 탐색합니다.

굴 탐험과 같습니다 — 갈림길이 나오면 한 길을 먼저 끝까지 들어가 보고, 막다른 길이면 뒤로 돌아와 다른 길을 탐색합니다.

1
현재 정점을 방문 처리한다
2
연결된 정점 중 아직 방문하지 않은 곳으로 들어간다
3
더 이상 갈 곳이 없으면 되돌아온다 (재귀 종료)
4
돌아온 자리에서 아직 안 간 다른 정점을 탐색한다
DFS가 잘 쓰이는 문제 유형
방문 순서 구하기 연결 요소 개수 경로 존재 여부 격자 영역 개수 사이클 탐지 기초 백트래킹 기초

"연결되어 있는가?", "영역의 개수?", "모든 경우 탐색?" → DFS 신호!

DFS 의사코드
DFS(현재정점): visited[현재정점] = 1 ← 들어오자마자 방문 처리 현재정점과 연결된 모든 next에 대해: if visited[next] == 0: DFS(next) ← 재귀로 더 깊이 들어감 ← 함수 끝나면 자동으로 되돌아옴

재귀 호출 = 다음 정점으로 이동. 함수가 끝나고 돌아오는 것 = 되돌아오기(백트래킹).

격자 DFS 핵심 — 방향 배열
int dx[4] = {-1, 1, 0, 0}; // 상, 하 int dy[4] = {0, 0, -1, 1}; // 좌, 우 void dfs(int x, int y) { visited[x][y] = 1; for (int dir = 0; dir < 4; dir++) { int nx = x + dx[dir]; int ny = y + dy[dir]; if (범위 초과) continue; if (벽) continue; if (방문함) continue; dfs(nx, ny); } }
재귀 DFS C코드 (기본)
#include <stdio.h> #define MAX 101 int graph[MAX][MAX]; int visited[MAX]; int n, m; void dfs(int now) { visited[now] = 1; // 들어오자마자 방문 처리 printf("%d ", now); for (int next = 1; next <= n; next++) { if (graph[now][next] == 1 && visited[next] == 0) { dfs(next); // 재귀 호출 = 더 깊이 } // 함수 끝 = 자동 되돌아옴 } } int main() { scanf("%d %d", &n, &m); for (int i = 0; i < m; i++) { int a, b; scanf("%d %d", &a, &b); graph[a][b] = graph[b][a] = 1; } dfs(1); return 0; }
연결 요소 개수 세기 C코드
int main() { int count = 0; /* 입력 생략 */ for (int i = 1; i <= n; i++) { if (visited[i] == 0) { // 아직 방문 안 한 정점 dfs(i); // 새 덩어리 탐색 시작 count++; // 연결 요소 1개 증가 } } printf("%d\n", count); }

"방문 안 한 정점에서 DFS 시작 = 새로운 연결 덩어리 하나 발견"

스택 DFS C코드 (재귀 없이)
void dfs_stack(int start) { int stack[MAX], top = -1; stack[++top] = start; while (top >= 0) { int now = stack[top--]; // 스택에서 꺼냄 if (visited[now]) continue; visited[now] = 1; printf("%d ", now); for (int next = n; next >= 1; next--) { if (graph[now][next] && !visited[next]) stack[++top] = next; } } }

재귀 DFS를 먼저 완전히 익히고 나서 스택 DFS로 넘어가는 것이 좋습니다.

재귀 / 스택 사용
연결 요소 · 경로 탐색
백트래킹 기초
영역 개수 세기
BFS (너비 우선)
가까운 곳부터 넓게
큐(Queue) 사용

문제 유형 구분 훈련
DFS "연결되어 있는가?", "영역 개수?", "모든 경우 탐색?", "덩어리 개수?"
BFS "최단거리?", "최소 이동 횟수?", "몇 단계 만에 도달?", "동시에 퍼짐?"

"끝까지 가 보는 구조?" → DFS / "최소 횟수?" → BFS — 이 구분 습관이 가장 중요합니다.

DFS에서 가장 많이 하는 실수 6가지
실수 1
방문 처리를 늦게 하는 실수
함수에 들어오자마자 visited[now] = 1을 해야 합니다. 늦게 하면 같은 정점을 여러 번 방문합니다.
실수 2
방문 배열 없이 재귀를 돌리는 실수
사이클이 있으면 무한 재귀가 발생합니다. visited 배열은 필수입니다.
실수 3
양방향 그래프를 한쪽만 연결하는 실수
무방향 그래프면 graph[a][b]와 graph[b][a] 둘 다 1로 설정해야 합니다.
실수 4
격자에서 범위 체크 누락
nx, ny가 배열 범위를 벗어나는지 먼저 확인해야 합니다. 범위 체크를 빼면 오류가 납니다.
실수 5
재귀 흐름을 머릿속으로 못 따라가는 실수
호출(들어감)과 복귀(되돌아옴)를 종이에 손으로 직접 써 보는 연습이 필수입니다.
실수 6
연결 요소 문제에서 전체 정점을 다 안 보는 실수
DFS 한 번으로 끝내면 안 됩니다. 1부터 n까지 모든 정점을 순회하며 미방문 정점에서 새로 DFS를 시작해야 합니다.
