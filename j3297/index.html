정올 #3297 구간의 합(PURQ) 1

Point Update와 Range Query를 펜윅 트리로 빠르게 처리하는 과정을 직접 확인합니다.

1. 문제를 쉽게 이해하기

Point Update

배열의 한 위치 값을 새 값으로 바꿉니다.

1 idx newValue

Range Query

배열의 left부터 right까지의 합을 구합니다.

2 left right

펜윅 트리

여러 칸의 합을 미리 묶어 저장해서 변경과 구간 합을 빠르게 처리합니다.

O(log N)

초등학생 버전: 숫자 카드가 아주 많이 줄 서 있습니다. 매번 하나씩 다 더하면 느리니, 몇 장씩 묶은 합을 미리 적어두는 방법이 펜윅 트리입니다.

4. 펜윅 트리 핵심 공식

lowbit

idx & -idx

idx가 담당하는 묶음 크기를 구합니다.

update 이동

idx += idx & -idx

값 변경이 영향을 주는 다음 묶음으로 이동합니다.

prefix_sum 이동

idx -= idx & -idx

앞에서부터 idx까지의 합을 만들기 위해 필요한 묶음을 거꾸로 더합니다.

예를 들어 idx=6이면 6의 이진수는 110이고, idx & -idx는 2입니다. 그래서 6번 tree는 2칸짜리 묶음을 담당한다고 생각할 수 있습니다.

5. 구간 합을 왜 prefix_sum으로 구할까?

left부터 right까지의 합은 다음처럼 구합니다.

range_sum(left, right) = prefix_sum(right) - prefix_sum(left - 1)
  1. prefix_sum(right)는 1번부터 right번까지의 합입니다.
  2. prefix_sum(left - 1)는 1번부터 left-1번까지의 합입니다.
  3. 둘을 빼면 left번부터 right번까지의 합만 남습니다.

6. 정답 C 코드

#include <stdio.h>

#define MAX 1000001

int N, M;
long long arr[MAX];
long long tree[MAX];

void update(int idx, long long diff) {
    while (idx <= N) {
        tree[idx] += diff;
        idx += idx & -idx;
    }
}

long long prefix_sum(int idx) {
    long long sum = 0;

    while (idx > 0) {
        sum += tree[idx];
        idx -= idx & -idx;
    }

    return sum;
}

long long range_sum(int left, int right) {
    return prefix_sum(right) - prefix_sum(left - 1);
}

int main() {
    int i;
    int cmd, a;
    long long b;

    scanf("%d", &N);

    for (i = 1; i <= N; i++) {
        scanf("%lld", &arr[i]);
        update(i, arr[i]);
    }

    scanf("%d", &M);

    for (i = 0; i < M; i++) {
        scanf("%d %d %lld", &cmd, &a, &b);

        if (cmd == 1) {
            long long diff = b - arr[a];
            arr[a] = b;
            update(a, diff);
        } else {
            printf("%lld\n", range_sum(a, (int)b));
        }
    }

    return 0;
}

7. 수업용 한 줄 정리

펜윅 트리는 “값 하나가 바뀌면 관련 묶음만 고치고, 구간 합은 필요한 묶음만 빠르게 더하는 자료구조”입니다.
