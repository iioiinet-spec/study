정올 #7017 두 배

숫자들을 작은 순서대로 줄 세우기 위해 숫자를 직접 키우지 않고, “몇 번 두 배 했는지”만 저장해서 빠르게 비교하는 알고리즘을 배웁니다.

문제 핵심 시간초과 코드 핵심 생각 비교 함수 이분 탐색 정답 코드 시뮬레이터 퀴즈

1. 문제 핵심

정올 #7017 “두 배”는 숫자들을 작은 순서대로 만들기 위해 각 숫자를 필요한 만큼 2배 하는 문제입니다.
1. 앞 숫자보다 현재 숫자가 작으면 현재 숫자를 2배 할 수 있습니다.
2. 실제 값을 계속 2배 하면 반복이 많아질 수 있습니다.
3. 큰 수가 되면 overflow 위험도 있습니다.
4. 그래서 실제 값 대신 두 배 횟수만 저장합니다.

1.1 한 줄 핵심

숫자를 직접 2배 하지 말고, “몇 번 2배 했는지”만 저장해서 비교한다.

2. 시간초과 나는 쉬운 코드

2.1 위험한 코드

while (cur < prev) {
    cur *= 2;
    cnt++;
}

이 코드는 숫자가 커질 때까지 계속 2배 합니다. 반복이 많아질 수 있고, cur가 너무 커져서 overflow가 날 수 있습니다.

2.2 왜 위험한가?

1. 반복 증가
필요한 두 배 횟수가 크면 while문이 오래 돕니다.
2. overflow
long long 범위를 넘어가면 값이 깨질 수 있습니다.
3. 큰 입력에서 위험
N이 크면 작은 반복도 전체로는 매우 커질 수 있습니다.

3. 핵심 생각

4. 실제로 곱하지 않고 비교하기

compare(a, ea, b, eb) 함수는 다음 두 수를 비교합니다.

a × 2^ea

b × 2^eb

4.1 숫자의 2진수 길이

bitlen(long long x)는 숫자가 2진수로 몇 자리인지 구합니다.

4.2 비교 흐름

bitlen(a)+ea 계산
→
bitlen(b)+eb 계산
→
자리수가 다르면 바로 비교
→
자리수가 같으면 shift로 맞춰 비교
compare 함수 코드 보기 
int bitlen(long long x) {
    int len = 0;

    while (x > 0) {
        len++;
        x /= 2;
    }

    return len;
}

int compare(long long a, int ea, long long b, int eb) {
    int la = bitlen(a) + ea;
    int lb = bitlen(b) + eb;

    if (la < lb) return -1;
    if (la > lb) return 1;

    if (ea > eb) {
        a <<= (ea - eb);
    } else {
        b <<= (eb - ea);
    }

    if (a < b) return -1;
    if (a > b) return 1;
    return 0;
}

5. 이분 탐색으로 필요한 두 배 횟수 찾기

현재 숫자를 몇 번 두 배 해야 앞 숫자 이상이 되는지 찾습니다.

0번, 1번, 2번을 하나씩 확인하지 않고 이분 탐색으로 빠르게 찾습니다.
int left = 0;
int right = prevExp + 40;

while (left < right) {
    int mid = (left + right) / 2;

    if (compare(a[i], mid, a[i - 1], prevExp) >= 0) {
        right = mid;
    } else {
        left = mid + 1;
    }
}

5.1 왜 시간초과가 안 나는가?

느린 방법
계속 2배하면서 확인
while 반복
빠른 방법
필요한 횟수를 이분 탐색으로 찾기
O(N log N)

6. 정답 C 코드

#include <stdio.h>

int bitlen(long long x) {
    int len = 0;

    while (x > 0) {
        len++;
        x /= 2;
    }

    return len;
}

int compare(long long a, int ea, long long b, int eb) {
    int la = bitlen(a) + ea;
    int lb = bitlen(b) + eb;

    if (la < lb) return -1;
    if (la > lb) return 1;

    if (ea > eb) {
        a <<= (ea - eb);
    } else {
        b <<= (eb - ea);
    }

    if (a < b) return -1;
    if (a > b) return 1;
    return 0;
}

int main() {
    int N;
    scanf("%d", &N);

    long long a[250001];

    for (int i = 0; i < N; i++) {
        scanf("%lld", &a[i]);
    }

    long long answer = 0;
    int prevExp = 0;

    for (int i = 1; i < N; i++) {
        int left = 0;
        int right = prevExp + 40;

        while (left < right) {
            int mid = (left + right) / 2;

            if (compare(a[i], mid, a[i - 1], prevExp) >= 0) {
                right = mid;
            } else {
                left = mid + 1;
            }
        }

        answer += left;
        prevExp = left;
    }

    printf("%lld\n", answer);

    return 0;
}

6.1 코드 작성 순서

1단계
bitlen(long long x)
숫자의 2진수 길이를 구합니다.
2단계
compare(a, ea, b, eb)
실제로 곱하지 않고 두 수를 비교합니다.
3단계
while (left < right)
현재 숫자를 몇 번 두 배 해야 하는지 이분 탐색합니다.
