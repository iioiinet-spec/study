순차 자료구조와 선형 리스트
// Sequential Data Structure & Linear List
자료구조(Data Structure)란 컴퓨터에서 자료를 효율적으로 저장·관리하기 위한 방법입니다. 어떤 자료구조를 선택하느냐에 따라 프로그램의 속도와 메모리 사용량이 크게 달라집니다.
📋 선형 리스트(Linear List) 핵심 개념
선형 리스트는 원소들 사이에 1:1 선형 관계가 있는 자료구조입니다. 원소들이 a₀, a₁, a₂, …, aₙ₋₁ 순서로 나열되며, 각 원소는 바로 앞·뒤 원소와만 연결됩니다.
⚡ 순차 자료구조 특징
📊 연산별 시간복잡도
메모리 저장 구조
// 순차 자료구조 = 연속된 메모리 공간에 저장
순차 자료구조에서 배열은 연속된 메모리 공간에 저장됩니다. 첫 번째 원소의 주소(base address)와 각 원소의 크기(element size)만 알면 어떤 원소든 O(1)에 바로 접근할 수 있습니다.
🗺️ 메모리 레이아웃 시각화 타입·크기별 비교
✅ 순차 구조 장점
    ❌ 순차 구조 단점
      삽입 연산 Insert
      // 지정한 위치에 새 원소를 삽입 — O(n)
      순차 리스트에서 k번 위치에 삽입하려면 k번부터 마지막 원소까지 모두 한 칸씩 뒤로 이동해야 합니다. 최악의 경우 n번 이동 → O(n).
      🎮 삽입 단계별 시뮬레이터
      📝 C 코드 — 삽입 함수
      삭제 연산 Delete
      // 지정한 위치의 원소를 삭제 — O(n)
      순차 리스트에서 k번 위치 원소를 삭제하면 k+1번부터 마지막 원소까지 모두 한 칸씩 앞으로 이동해야 합니다. 최악의 경우 n번 이동 → O(n).
      🎮 삭제 단계별 시뮬레이터
      📝 C 코드 — 삭제 함수
      탐색 연산 Search
      // 원하는 값 찾기 — 순차 탐색 O(n) / 이진 탐색 O(log n)
      순차 탐색(Sequential Search)
      앞에서부터 하나씩 비교. 정렬 불필요. 평균 n/2번, 최악 n번 비교 → O(n)
      이진 탐색(Binary Search)
      정렬된 배열에서만 가능. 범위를 절반씩 축소. 최악 log₂n번 비교 → O(log n)
      🎮 탐색 시뮬레이터 순차 / 이진 비교
      순차 탐색 로그
      이진 탐색 로그

      다항식 배열 표현
      // 순차 자료구조로 다항식(polynomial) 저장하기
      다항식 P(x) = aₙxⁿ + aₙ₋₁xⁿ⁻¹ + … + a₁x + a₀ 를 배열로 표현할 때, 인덱스 i에 xⁱ의 계수를 저장합니다. 예: arr[3] = 5 → 5x³ 항.

      행렬 2차원 배열 표현
      // 2D Array — 행(row) × 열(column) 순서로 저장
      2차원 배열 A[m][n]은 m행 n열 행렬입니다. C언어는 행 우선(Row-Major) 순서로 메모리에 저장합니다. A[i][j]의 주소 = base + (i×n + j) × size.
      🎮 행렬 클릭 & 주소 계산기 셀을 클릭하세요
      셀을 클릭하면 주소가 계산됩니다
      ■ 선택 행 ■ 선택 열 ■ 선택 셀
      🔄 행 우선 저장 순서 시각화
      * C언어: A[0][0]→A[0][1]→A[0][2]→A[1][0]→… (행 우선)
      * Fortran/MATLAB: A[0][0]→A[1][0]→A[2][0]→A[0][1]→… (열 우선)
      주소 계산 공식
      // 배열 원소의 메모리 주소를 직접 계산
      1️⃣ 1차원 배열 주소 계산기
      공식: addr(A[i]) = base + i × size
      2️⃣ 2차원 배열 주소 계산기
      공식(행 우선): addr(A[i][j]) = base + (i×n + j) × size
