정올 #5125 피하자

홀수와 짝수가 서로 섞여 있으면 불편해요. 최소 몇 번 옆으로 바꿔야 한쪽에는 짝수, 한쪽에는 홀수가 모일까요?

1. 문제를 아주 쉽게 이해하기

숫자들이 한 줄로 서 있습니다. 우리는 바로 옆에 있는 숫자끼리만 자리를 바꿀 수 있습니다. 목표는 홀수와 짝수가 만나는 부분이 최대 1번만 나오게 만드는 것입니다.

가능한 모양 1

짝수를 왼쪽으로 모읍니다.

4 0 5 1

가능한 모양 2

홀수를 왼쪽으로 모읍니다.

5 1 4 0

0은 짝수입니다. 따라서 0은 파란색 짝수 그룹에 들어갑니다.

2. 직접 입력해서 확인하기

숫자를 띄어쓰기로 입력하세요. 예: 4 5 1 0

입력한 숫자

짝수를 왼쪽으로 모으기

현재 짝수 앞에 있는 홀수 개수만큼 이동합니다.

0

홀수를 왼쪽으로 모으기

현재 홀수 앞에 있는 짝수 개수만큼 이동합니다.

0

정답

두 방법 중 더 적은 횟수입니다.

0

정렬 결과 보기

짝수 왼쪽 결과

홀수 왼쪽 결과

3. 초등학생도 이해하는 풀이 원리

  1. 숫자를 왼쪽부터 하나씩 봅니다.
  2. 짝수를 왼쪽으로 모으고 싶다면, 짝수 앞에 있는 홀수들이 방해물입니다.
  3. 현재 짝수 앞에 홀수가 2개 있으면, 그 짝수는 홀수 2개를 지나가야 하므로 2번 움직입니다.
  4. 반대로 홀수를 왼쪽으로 모으고 싶다면, 홀수 앞에 있는 짝수들이 방해물입니다.
  5. 두 경우를 모두 계산해서 더 작은 값을 정답으로 출력합니다.

4. 예제 설명

입력 숫자가 다음과 같다고 해봅시다.

4 5 1 0

마지막 0은 짝수입니다. 짝수를 왼쪽으로 모으려면 0이 앞의 홀수 5와 1을 지나가야 합니다.

그래서 2번 움직입니다.

4 5 1 0
4 5 0 1
4 0 5 1

정답: 2

5. 정답 C 코드

#include <stdio.h>

int main() {
    int N;
    long long even_count = 0;
    long long odd_count = 0;
    long long even_left = 0;
    long long odd_left = 0;
    long long x;
    int i;

    scanf("%d", &N);

    for (i = 0; i < N; i++) {
        scanf("%lld", &x);

        if (x % 2 == 0) {
            // 짝수를 왼쪽으로 모으려면,
            // 앞에 있던 홀수들을 모두 지나가야 한다.
            even_left += odd_count;
            even_count++;
        } else {
            // 홀수를 왼쪽으로 모으려면,
            // 앞에 있던 짝수들을 모두 지나가야 한다.
            odd_left += even_count;
            odd_count++;
        }
    }

    if (even_left < odd_left) {
        printf("%lld\n", even_left);
    } else {
        printf("%lld\n", odd_left);
    }

    return 0;
}
