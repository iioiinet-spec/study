펜윅 트리 Fenwick Tree / BIT

한 위치 값 변경과 구간 합을 빠르게 처리하는 자료구조를 배열, 묶음 합, 인덱스 이동으로 이해합니다.

1. 펜윅 트리란?

빠른 값 변경

특정 위치 값이 바뀌면 관련된 묶음 합만 고칩니다.

update(idx, diff)

빠른 누적합

1번부터 idx번까지의 합을 묶음 몇 개로 빠르게 구합니다.

prefix_sum(idx)

빠른 구간합

left부터 right까지의 합은 누적합 2개를 빼서 구합니다.

range_sum(l, r)

초등학생 버전: 숫자 카드가 많을 때 매번 하나씩 더하면 느립니다. 그래서 1장, 2장, 4장, 8장처럼 묶음 합을 미리 적어두고 필요한 묶음만 빠르게 더합니다.

4. tree[idx]가 담당하는 구간

펜윅 트리의 각 칸은 배열의 특정 구간 합을 저장합니다.

tree[idx] = arr[idx - lowbit(idx) + 1] + ... + arr[idx]

5. 구현 순서

  1. 배열은 1번 인덱스부터 사용합니다.
  2. tree 배열을 만들고 모두 0으로 시작합니다.
  3. 초기 값을 넣을 때마다 update(i, arr[i])를 호출합니다.
  4. 값을 바꿀 때는 새 값과 기존 값의 차이 diff를 계산합니다.
  5. update에서는 idx += idx & -idx로 이동합니다.
  6. prefix_sum에서는 idx -= idx & -idx로 이동합니다.
  7. 구간 합은 prefix_sum(right) - prefix_sum(left - 1)입니다.

6. 전체 C 코드

#include <stdio.h>

#define MAX 1000001

int N;
long long arr[MAX];
long long tree[MAX];

void update(int idx, long long diff) {
    while (idx <= N) {
        tree[idx] += diff;
        idx += idx & -idx;
    }
}

long long prefix_sum(int idx) {
    long long sum = 0;

    while (idx > 0) {
        sum += tree[idx];
        idx -= idx & -idx;
    }

    return sum;
}

long long range_sum(int left, int right) {
    return prefix_sum(right) - prefix_sum(left - 1);
}

int main() {
    int i;

    scanf("%d", &N);

    for (i = 1; i <= N; i++) {
        scanf("%lld", &arr[i]);
        update(i, arr[i]);
    }

    printf("1~N 합 = %lld\n", prefix_sum(N));
    printf("2~4 합 = %lld\n", range_sum(2, 4));

    return 0;
}

7. 핵심 한 줄 정리

펜윅 트리는 “배열의 합을 여러 묶음으로 저장해 두고, 값 변경과 구간 합을 logN번 이동으로 해결하는 방법”입니다.
