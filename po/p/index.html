플래티넘 1. 다항식 빠른 계산

다항식의 계수가 낮은 차수부터 주어집니다.

P(x) = a₀ + a₁x + a₂x² + ... + aₙxⁿ

Horner 알고리즘은 이 식을 다음처럼 바꿔 계산합니다.

P(x) = (((aₙx + aₙ₋₁)x + aₙ₋₂) ... x + a₀)

높은 차수 계수부터 시작해서, 매번 “곱하고 더하기”만 반복합니다.

조립제법과 나머지 정리

Horner 알고리즘은 단순히 값을 빠르게 계산하는 방법이 아니라, 조립제법과도 연결됩니다.

P(x)를 x - K로 나눈 나머지 = P(K)

따라서 Horner 계산의 마지막 결과는 P(K)이면서, 동시에 x-K로 나누었을 때의 나머지입니다.

1. 현재 다항식

2. 조립제법 표

높은 차수 계수부터 내려오면서 계산합니다.

3. 의미

예를 들어 P(x)를 x-2로 나눌 때, 나머지는 P(2)입니다.

즉, 문제에서 K=2라면 Horner 알고리즘으로 구한 값은 곧 나머지입니다.

왜 플래티넘 난이도인가?

입력 크기 N이 매우 크면, 단순한 거듭제곱 계산이나 비효율적인 방식은 위험합니다.

방법 일반적인 위험한 방식 Horner 방식
계산 형태 aᵢ × Kⁱ를 직접 계산 result = result × K + aᵢ
오버플로우 위험 매우 큼 매 단계 MOD 처리 가능
구현 난이도 쉬워 보이지만 실수 많음 반복 구조가 깔끔함
시간복잡도 잘못 구현하면 O(N²) 가능 O(N)

N ≤ 1,000,000 같은 큰 입력에서는 한 번의 반복으로 끝나는 O(N) 알고리즘이 필요합니다.

연산 횟수 비교 체험

C 정답 코드

#include <stdio.h>

#define MOD 1000000007

long long coef[1000001];

int main(void)
{
    int N;
    long long K;

    scanf("%d", &N);

    for(int i = 0; i <= N; i++)
    {
        scanf("%lld", &coef[i]);
        coef[i] %= MOD;
    }

    scanf("%lld", &K);
    K %= MOD;

    long long result = coef[N];

    for(int i = N - 1; i >= 0; i--)
    {
        result = (result * K + coef[i]) % MOD;
    }

    printf("%lld", result);

    return 0;
}
