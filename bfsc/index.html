완성된 BFS 코드 전체
#include <stdio.h> #define MAX 101 // ① 그래프, 방문, 큐 배열 int graph[MAX][MAX]; // 인접 행렬 int visited[MAX]; // 방문 여부 int queue[MAX]; // BFS 큐 int front = 0, rear = 0; // 큐 앞뒤 포인터 int n, m; void bfs(int start) { // ② 시작점 처리 queue[rear++] = start; visited[start] = 1; // ③ 큐가 빌 때까지 반복 while (front < rear) { // ④ 앞에서 하나 꺼내기 int now = queue[front++]; printf("%d ", now); // ⑤ 연결된 정점 확인 (작은 번호부터) for (int next = 1; next <= n; next++) { if (graph[now][next] == 1 && visited[next] == 0) { visited[next] = 1; // ⑥ 방문 처리 queue[rear++] = next; // ⑦ 큐에 삽입 } } } } int main() { scanf("%d %d", &n, &m); for (int i = 0; i < m; i++) { int a, b; scanf("%d %d", &a, &b); // ⑧ 무방향: 양쪽 다 저장 graph[a][b] = 1; graph[b][a] = 1; } bfs(1); return 0; }

7개 핵심 포인트: ①배열 선언 → ②시작점 삽입 → ③while 반복 → ④꺼내기 → ⑤연결 확인 → ⑥방문 처리 → ⑦삽입. 이 순서가 BFS의 뼈대입니다.

입출력 예제
입력
5 5 1 2 1 3 2 4 2 5 3 5
출력
1 2 3 4 5
이 문제에서 자주 하는 실수 4가지
실수 1
무방향 그래프를 한쪽만 저장
graph[a][b]=1만 하고 graph[b][a]=1을 빠뜨리면 탐색이 제대로 안 됩니다.
실수 2
방문 처리를 늦게 하는 실수
큐에 넣을 때 즉시 visited[next]=1을 해야 합니다. 꺼낼 때 하면 같은 정점이 큐에 여러 번 삽입될 수 있습니다.
실수 3
front, rear 혼동
rear는 넣는 쪽(뒤), front는 꺼내는 쪽(앞)입니다. front < rear면 큐에 원소가 있습니다.
실수 4
작은 번호부터 방문 조건 누락
for(next=1; next<=n; next++)로 순서대로 확인해야 번호가 작은 정점부터 방문합니다.
이 문제에서 확장할 수 있는 것
시작 정점을 입력으로 받게 바꾸기
방문 순서 대신 각 정점까지 거리 출력하기 → dist[next]=dist[now]+1
연결 요소 개수 세기 → 미방문 정점마다 BFS 시작
